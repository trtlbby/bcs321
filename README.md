This repository contains my academic logs and solutions for BCS 321: Programming Languages Design and Implementation. It serves as a record of my assignments, exercises, and projects focused on parsing, analyzing, and manipulating programming language constructs using Java.

## Purpose

- **Personal log:** Track my coursework progress and revisit my solutions for future review.

## Project Structure

The repository is organized by task and activity (AS OF THE MOMENT):
- `TASKC/`  
  - `taskC_pretest_bacsain_earllawrence.java`: Handles file I/O, string manipulation, word classification (numbers, identifiers, operators), detection of reserved keywords, and error logging for invalid tokens.
  - `input.txt`: Example input file with assignment statements for parsing and analysis.
- `TASKD/`
  - `inclass01a_bacsain_earllawrence.java`: Checks if user input is a valid identifier based on defined rules.
  - `inclass01b_bacsain_earllawrence.java`: Parses log strings to extract date, time, and error levels.

## Main Features TODO at the end of Sem

- **Lexical Analysis:**  
  - Splits input strings/statements into words/tokens.
  - Classifies tokens as numbers, identifiers, or operators.
  - Checks tokens against a list of reserved keywords.

- **Error Detection & Logging:**  
  - Identifies invalid or unknown tokens while processing code statements.
  - Collects and reports errors not matching valid patterns or reserved keywords.

- **Identifier Validation:**  
  - Ensures user input meets the requirements for programming language identifiers (e.g., length, starting character, allowed characters).

- **Log Parsing:**  
  - Extracts structured information (date, time, log level) from formatted log strings.
